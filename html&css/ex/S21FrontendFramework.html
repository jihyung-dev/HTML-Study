프론트엔드 프레임워크와 부트스트랩

1. 프레임워크(Framework)란?

  • 프레임워크는 소프트웨어 개발 시 필요한 기본 틀을 미리 만들어 제공하는 도구
  • 개발자는 일일이 모든 기능을 구현하지 않고, 제공되는 규칙과 구조에 맞추어 개발
  • “새로운 틀을 제공하여 새로운 규칙대로 작업하는 것” → 생산성과 유지보수 향상

2. 웹 프레임워크

  • 웹 애플리케이션 제작에 필요한 기능(라우팅, UI, 데이터 처리 등)을 제공
  • 개발자는 핵심 로직에 집중하고, 공통 기능은 프레임워크 규칙대로 활용
  • 대표적인 웹 프레임워크:
    • 프론트엔드: Bootstrap, Tailwind CSS, Foundation
    • 백엔드: Django, Spring, Ruby on Rails, Express.js

3. 반응형 웹앱 프레임워크

  • 다양한 화면 크기(모바일, 태블릿, PC)에 따라 자동으로 레이아웃이 변하는 웹앱 제작 지원
  • CSS의 미디어쿼리를 기반으로 레이아웃을 유연하게 조정
  • 대표적인 반응형 프레임워크:
    • Bootstrap
    • Foundation
    • Bulma

4. Bootstrap 소개

  • 역사 : 2010년 Twitter 개발자들이 내부 툴을 만들기 위해 시작 → 2011년 오픈소스로 공개 → 현재까지 가장 인기 있는 프론트엔드 프레임워크
  • CSS/JS 기반의 프론트엔드 프레임워크로, 반응형 웹 구현에 최적화
  • 버튼, 네비게이션 바, 모달 창, 카드 등 자주 쓰이는 UI 컴포넌트를 클래스 형태로 제공
  • 커스터마이징 가능하며, 필요한 부분만 선택적으로 사용할 수 있음
  • 공식 사이트: https://getbootstrap.com/

5. Bootstrap 동작 원리 (클래스 기반)

  • HTML 요소에 특정한 클래스명을 부여하면, Bootstrap의 CSS 규칙이 자동 적용됨
  • 예:
    • <button class="btn btn-primary">확인</button>
    • → btn : 버튼 기본 스타일 적용
    • → btn-primary : 파란색 주요 버튼 스타일 적용
  • Grid System(그리드 시스템)을 통해 화면 크기에 따라 자동으로 레이아웃을 조정
  • JS 플러그인을 포함해 모달, 드롭다운, 캐러셀 같은 동적 UI도 구현 가능

6. Bootstrap 적용 방법

  • Bootstrap을 프로젝트에 적용하는 방법은 크게 두 가지
  • 1) 파일 다운로드 방식
    • 공식 사이트에서 CSS/JS 파일을 내려받아 프로젝트 폴더에 저장
    • HTML에서 <link>와 <script>로 직접 불러오기
    • 인터넷 연결이 없어도 사용 가능
  • 2) CDN(Content Delivery Network) 방식
    • Bootstrap 파일을 직접 저장하지 않고, 전 세계 서버에 배포된 파일을 URL로 불러옴
    • 공식 제공 CDN 주소를 HTML에 삽입하면 바로 사용 가능
    • 최신 버전을 쉽게 적용할 수 있고, 빠른 로딩 속도 장점
  • 3) npm 또는 yarn 방식
    • npm(Node Package Manager) : Node.js에 기본 포함된 패키지 매니저
    • yarn : 페이스북에서 만든 또 다른 패키지 매니저(속도와 안정성 향상)
    • 명령어 예시: 
      npm install bootstrap
yarn add bootstrap
    • 설치 후 node_modules 폴더에 Bootstrap이 포함되고, import 문법으로 JS/CSS를 불러와 사용
    • 예시 (ES Module 방식): 
      import 'bootstrap/dist/css/bootstrap.min.css';
import 'bootstrap/dist/js/bootstrap.bundle.min.js';

7. CDN(Content Delivery Network)란?

  • 정적 파일(CSS, JS, 이미지 등)을 전 세계 여러 서버에 분산 배치해 사용자에게 가장 가까운 서버에서 제공하는 방식
  • 웹사이트 속도를 빠르게 하고, 서버 부담을 줄이는 효과
  • Bootstrap 같은 라이브러리는 CDN 방식을 많이 사용
  • 예시 (Bootstrap v5.3): 
    <!-- Bootstrap CSS -->
<link
  href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/css/bootstrap.min.css"
  rel="stylesheet">

<!-- Bootstrap JS (팝업/모달 등 동적 기능용) -->
<script
  src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/js/bootstrap.bundle.min.js">
</script>

9. JavaScript 패키지 매니저란?

  • JavaScript에서 사용하는 외부 라이브러리나 프레임워크를 손쉽게 설치, 관리해주는 도구
  • 대표적인 패키지 매니저: npm, yarn, pnpm
  • 기능:
    • 외부 라이브러리(예: Bootstrap, React, jQuery)를 자동 설치
    • 버전 관리: 특정 버전 고정, 업데이트 용이
    • 의존성 관리: 라이브러리들이 필요로 하는 다른 라이브러리도 함께 설치
  • 왜 필요한가?
    • 수동으로 파일을 내려받아 붙이는 방식은 번거롭고, 버전 충돌 위험 ↑
    • 패키지 매니저를 사용하면 한 줄 명령어로 설치 및 업데이트 가능
    • 팀 협업 시 package.json 파일만 공유하면, 누구든 같은 환경을 쉽게 재현 가능

1. npm (Node Package Manager)

  • 등장: 2010년 (Node.js와 함께 기본 패키지 매니저로 제공)
  • 장점:
    • 가장 오래되고 널리 사용됨 → 생태계가 가장 큼
    • Node.js 설치 시 자동 포함 → 별도 설치 필요 없음
    • npm registry에 수백만 개의 라이브러리 제공
  • 단점:
    • 과거에는 속도가 느리고 의존성 관리가 비효율적이라는 평가가 있었음
    • 버전 충돌 문제로 인한 "dependency hell" 이슈 경험 많음 (최근 개선됨)

2. Yarn

  • 등장: 2016년 (Facebook이 npm의 단점을 보완하기 위해 개발)
  • 장점:
    • npm보다 빠른 속도 (캐시와 병렬 다운로드)
    • lock 파일(yarn.lock)로 의존성 버전을 엄격히 고정 → 협업 시 동일 환경 재현 용이
    • 안정성과 보안 강화 (checksum 검증)
  • 단점:
    • npm보다 생태계 규모가 작음 (하지만 호환성은 높음)
    • 추가 설치 필요 (Node.js에 기본 포함 아님)

3. pnpm

  • 등장: 2017년 (npm과 yarn의 단점을 보완하며 가볍고 빠른 대안으로 등장)
  • 장점:
    • 디스크 공간 절약: 공통 의존성을 하드 링크로 관리 → node_modules 중복 방지
    • 속도가 매우 빠름 (효율적인 캐싱 구조)
    • 대규모 프로젝트/모노레포(monorepo) 관리에 강점
  • 단점:
    • npm, yarn보다 상대적으로 신생 → 기업 채택률은 낮은 편
    • 특정 툴/플러그인에서 호환성 이슈가 간혹 발생

부트스트랩과제

과제 1: 버튼 & 배지 연습

다양한 버튼 변형을 만들고 배지를 버튼에 겹쳐 배치해보세요.

  • btn 변형 5가지 이상(primary, outline, disabled 포함)
  • 버튼 오른쪽 상단에 badge 표시 (position 유틸 사용)
  • Hover 시 shadow 유틸로 그림자 효과 적용

과제 2: 알림(Alerts) 변형

부트스트랩 alert 컴포넌트를 활용하여 알림 UI를 다양하게 구성하세요.

  • alert 색상 변형 4종 이상
  • alert-dismissible 닫기 버튼 활성화
  • alert-link 스타일 포함
  • 상단 고정(alert를 position-fixed로 배치)

과제 3: 토스트(Toast) 알림

버튼 클릭 시 토스트가 표시되도록 구현하세요.

  • 우하단에 스택되는 토스트 2개 이상
  • 버튼 클릭으로 토스트 표시
  • autohide와 지연 시간 옵션 다르게 적용

과제 4: 모달(Modal) & 이미지 프리뷰

이미지 썸네일을 클릭하면 모달로 큰 이미지가 보이도록 구성하세요.

  • 모달 헤더/바디/푸터 구분
  • fade 전환 효과와 modal-dialog-centered 적용
  • ESC 키로 닫기 가능

과제 5: 오프캔버스(Offcanvas) 사이드 메뉴

좌측에서 열리는 사이드 메뉴를 만들어보세요.

  • 메뉴/설정/로그아웃 항목 리스트
  • form-switch 토글 추가(다크모드 스위치처럼)
  • 열고 닫을 때 body 스크롤 동작 확인

과제 6: 콜랩스/아코디언(Collapse/Accordion)

FAQ 아코디언을 구성해보세요.

  • FAQ 3문항 구성
  • 한 번에 하나만 열리도록 설정
  • 아이콘이 펼칠 때 회전(transition, transform 유틸 활용)

과제 7: 프로그레스바 & 스피너

다운로드 진행률과 로딩 상태를 표현해보세요.

  • progressbar 0→100% 버튼 클릭으로 10%씩 증가
  • 진행 중에는 spinner-border 표시, 완료 시 숨김
  • aria-* 속성으로 접근성 정보 제공

과제 8: 폼(Form) 검증

회원가입 폼을 만들고 유효성 검증을 구현하세요.

  • 이메일/비밀번호/동의 체크 구성
  • is-valid / is-invalid 상태 표시
  • valid-feedback / invalid-feedback 메시지 표시

과제 9: 리스트 그룹(List Group) + 드롭다운

알림 센터 형태의 리스트 그룹을 구성하세요.

  • 읽음/안읽음 스타일 구분(opacity-50 등)
  • 각 항목에 dropdown 버튼으로 “읽음 처리/삭제” 메뉴
  • 키보드 접근성 확인

과제 10: 카루셀(Carousel) 슬라이드

이미지와 캡션이 있는 슬라이드를 구성하세요.

  • 이미지 3장과 캡션 추가
  • 자동 재생, 좌우 컨트롤, 인디케이터 활성화
  • 작은 화면에서는 캡션 숨김(d-none d-sm-block)

과제 11: 유틸리티 챌린지

텍스트, 색상, 간격 유틸리티만으로 콘텐츠를 꾸며보세요.

  • text-* 유틸로 제목/본문/보조 텍스트 대비
  • spacing 유틸만으로 타이포 간격 조정
  • bg-*와 텍스트 대비 고려
  • 다크 배경에서 링크 hover/포커스 스타일 구분
