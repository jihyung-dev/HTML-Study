CSS의 단위에 대해 알아봅시다.

cm,m,km,원,도씨와 같은 일상적인 단위는 css에서 존재하지 않는다.

브라우저(css)에서 사용하는 단위들..

  • px :
    100px*100px
    모니터(디스플레이)의 해상도 기준 크기
  • 30%*30%
  • vw vh (viewport 브라우저 창 크기 기준 100~0) :
  • rem [절대기준] (1rem == 16px(기본 font 사이즈))
    글자크기를 기준으로 크기지정(**)
  • em [상대기준] (시작은 16px == 1em, 부모요소의 크기를 기준으로 계속 바뀜)
    10em*10em
    부모의 폰트사이즈를 em으로 변경하면 자식은 부모가 변경한 font-size를 1em으로 보고 계산한다.

  • 색에서 사용하는 단위

    • rgb(0~255,0~255,0~255)
      rgb(255,0,0)
    • rgba(0~255,0~255,0~255,0~1)
      rgb(255,0,0)
      투명도가 있는 색
    • 16진수 색 #(00~FF == 0~255,00~ff,00~ff) :
    • HSL (색조 Hue 0~360, 채도 Saturation(0~100), 명도 Lightness(0~100) :
      hsl(200,70%,70%)
    • HSLA (색조 Hue 0~360, 채도 Saturation (0~100), 명도 Lightness(0~100), 0~1(Alpha) :
      hsl(200,70%,70%,.5)

    단위 작성시 주의할 점

    • 0(%, px, em 모두 없다는 뜻)을 제외하고 단위가 없는 수는 없다.
    • 1000*1000
      0(%,px,em 모두 없다는 뜻)을 제외하고 단위가 없는 수는 없다.
    • 1000px+10px
      계산으로 값을 대입할 수 없다.
    • calc(300px/4);
      계산할 때는 calc를 사용
    • calc(300px+100px);
      calc 연산시 연산자 주변에 공백을 줘야 오류가 발생하지 않음
