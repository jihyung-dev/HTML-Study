자바스크립트 함수 function

함수(Function)란?

  • 하나의 특별한 작업(기능)을 수행하도록 작성된 코드 블록
  • 여러 번 반복해서 사용할 수 있도록 연산을 재사용하는 것이 목적
  • 입력(매개변수)을 받아 처리한 후 출력(반환값)을 돌려줄 수 있다
  • 코드를 분리해 프로그램을 더 이해하기 쉽고 유지보수하기 쉽게 만든다

함수의 구조

function 함수이름(매개변수1, 매개변수2, ...) {
  // 실행할 코드
  return 반환값; // (선택 사항)
}
  • 함수 이름: 함수를 구분하기 위한 식별자
  • 매개변수(Parameter): 함수에 전달되는 입력 값
  • 함수 본문: 실제 실행할 코드 블록
  • return: 함수 실행 결과를 반환 (없으면 undefined 반환)

함수 작성법

1. 함수 선언문 (Function Declaration)

function add(a, b) {
  return a + b;
}
console.log(add(2, 3)); // 5

2. 함수 표현식 (Function Expression)

const multiply = function(a, b) {
  return a * b;
};
console.log(multiply(2, 3)); // 6
  • 변수에 함수를 대입하는 방식
  • 함수를 값처럼 다룰 수 있다

3. 화살표 함수 (Arrow Function, ES6 이후)

화살표 함수는 프로토타입과 this바인딩 관련되어 다시 다룰 예정

// 할당 전에 호출하면 에러 발생
sayBye(); // ❌ ReferenceError

const sayBye = () => {
  console.log("잘 가!");
};

→ 화살표 함수도 함수 표현식이므로, 할당되기 전에는 사용할 수 없다.

함수의 호이스팅(Hoisting)

  • 호이스팅이란, 코드 실행 전 자바스크립트 엔진이 변수와 함수 선언을 메모리에 먼저 올려두는 것을 말한다.
  • 그래서 함수 선언문은 코드 작성 위치와 상관없이 미리 사용할 수 있다.
  • 반대로, 함수 표현식은 변수에 함수가 할당되는 시점에야 사용할 수 있다.

함수 호이스팅의 의도

  • 실행 컨텍스트는 개발자가 함수를 어디에 선언해도 미리 사용할 수 있도록 일부러 함수 선언문을 메모리에 올려둔다.
  • 이 덕분에 코드 순서와 관계없이 함수를 호출할 수 있어 편리하다.

최근에는 함수 표현식 선호

  • 프로그램이 복잡해질수록, 모든 함수를 미리 메모리에 올려두면 불필요한 메모리 사용이 생길 수 있다.
  • 함수 표현식(특히 const로 선언)은 필요할 때만 함수가 메모리에 만들어지기 때문에 더 안전하고 명확하다.
  • 그래서 최근 자바스크립트 스타일 가이드에서는 함수 표현식을 더 많이 사용한다.

예시

1. 함수 선언문 (Function Declaration)

// 선언보다 먼저 호출 가능
sayHello();

function sayHello() {
  console.log("안녕하세요!");
}

→ 함수 선언 전체가 호이스팅되므로 정상적으로 실행된다.

2. 함수 표현식 (Function Expression)

// 할당 전에 호출하면 에러 발생
sayHi(); // ❌ TypeError

const sayHi = function() {
  console.log("안녕!");
};

→ 호출 시점에는 아직 함수가 없음.

함수를 const로 선언하는 이유

자바스크립트는 타입이 안전하지 않은 언어입니다. 때문에 객체나 함수를 선언시 상수를 선호합니다.

재할당 시 생기는 문제 (예시 코드)

// 함수 선언 (var, let 사용 시)
let sayHi = function() {
  console.log("안녕하세요!");
};

// 실수로 재할당
sayHi = "다른 값이 들어옴";

sayHi(); // ❌ TypeError: sayHi is not a function

// const 사용 시
const sayHello = function() {
  console.log("안녕하세요!");
};

// 재할당 시도
sayHello = "다른 값"; // ❌ TypeError: Assignment to constant variable.

기명 함수 표기법 (Named Function Expression): 더이상 사용하지 않음

기명 함수 표기법은 디버깅을 위해 주로 사용했지만 엔진의 발달로 함수표현시의 이름이 출력되고, 재귀함수 사용을 권장하지 않아 더이상 사용되지 않음

  • 정의: 함수 표현식에 이름을 붙여주는 방식
  • 보통 함수 표현식은 이름이 없지만(익명 함수), 이름을 붙이면 함수 내부에서 자기 자신을 참조할 수 있다.(재귀함수)
  • 디버깅 시 스택 추적에 함수 이름이 표시되어 에러 원인을 찾기 쉽다.

예시

// 익명 함수 표현식
const sayHi = function() {
  console.log("안녕");
};

// 기명 함수 표현식
const sayHello = function greet() {
  console.log("Hello");
  // 자기 자신을 참조 가능
  greet(); // 재귀 호출 시 사용 가능
};

특징

  • 함수 이름은 함수 내부에서만 쓸 수 있고, 외부에서는 사용할 수 없다.
  • 재귀 호출이나 디버깅(스택 트레이스)에 유용하다.
  • 실무에서는 자주 쓰이지 않지만, 개념을 알아두면 도움이 된다.

함수 실습 문제 (중복 없는, 기초 심화)

  • 문제 1 — 문자열 길이 판별
    문자열을 받아 길이가 5 이상이면 "long", 아니면 "short"를 반환하는 함수 checkLength(str)를 작성하라.
  • 문제 2 — 첫 글자 비교
    두 문자열을 받아, 첫 글자가 같으면 true, 다르면 false를 반환하는 함수 sameFirstChar(a, b)를 작성하라.
  • 문제 3 — 배열의 첫 번째와 마지막 비교
    배열을 받아 첫 번째 요소와 마지막 요소가 같으면 true, 다르면 false를 반환하는 함수 isSameEdge(arr)를 작성하라.
  • 문제 4 — 짝수 합 / 홀수 합 구분
    숫자 두 개를 받아 짝수끼리 합하면 그 합을, 홀수끼리 합하면 그 합을, 짝홀 섞이면 0을 반환하는 함수 sumEvenOdd(a, b)를 작성하라.
  • 문제 5 — 특정 위치 문자 추출
    문자열과 인덱스를 받아, 해당 위치의 문자를 반환하는 함수 charAtIndex(str, i)를 작성하라. 단, 인덱스가 범위를 벗어나면 "없음"을 반환한다.
  • 문제 6 — 배열 길이 비교
    배열 두 개를 받아, 더 긴 배열을 반환하는 함수 longerArray(a, b)를 작성하라.
  • 문제 7 — 문자열 끝 확인
    문자열과 한 문자를 받아, 해당 문자열이 그 문자로 끝나면 true, 아니면 false를 반환하는 함수 endsWithChar(str, ch)를 작성하라.
  • 문제 8 — 양수/음수/0 판별
    정수를 받아, 양수면 "positive", 음수면 "negative", 0이면 "zero"를 반환하는 함수 checkSign(n)을 작성하라.
  • 문제 9 — 같은 타입인지 판별
    두 값을 받아, 같은 자료형이면 true, 다르면 false를 반환하는 함수 sameType(a, b)를 작성하라.
  • 문제 10 — 문자열 자릿수 비교
    숫자 두 개를 받아, 각 숫자를 문자열로 바꾸어 길이를 비교하고 더 긴 자릿수를 가진 숫자를 반환하는 함수 longerDigit(a, b)를 작성하라. 예) longerDigit(15, 300) → 300

함수 실습 문제 정답

  • 문제 1 — 문자열 길이 판별 
    function checkLength(str) {
  if (str.length >= 5) {
    return "long";
  } else {
    return "short";
  }
}
  • 문제 2 — 첫 글자 비교 
    function sameFirstChar(a, b) {
  return a[0] === b[0];
}
  • 문제 3 — 배열의 첫 번째와 마지막 비교 
    function isSameEdge(arr) {
  if (arr.length === 0) return false;
  return arr[0] === arr[arr.length - 1];
}
  • 문제 4 — 짝수 합 / 홀수 합 구분 
    function sumEvenOdd(a, b) {
  if (a % 2 === 0 && b % 2 === 0) {
    return a + b;
  } else if (a % 2 !== 0 && b % 2 !== 0) {
    return a + b;
  } else {
    return 0;
  }
}
  • 문제 5 — 특정 위치 문자 추출 
    function charAtIndex(str, i) {
  if (i < 0 || i >= str.length) {
    return "없음";
  }
  return str[i];
}
  • 문제 6 — 배열 길이 비교 
    function longerArray(a, b) {
  if (a.length >= b.length) {
    return a;
  } else {
    return b;
  }
}
  • 문제 7 — 문자열 끝 확인 
    function endsWithChar(str, ch) {
  return str[str.length - 1] === ch;
}
  • 문제 8 — 양수/음수/0 판별 
    function checkSign(n) {
  if (n > 0) {
    return "positive";
  } else if (n < 0) {
    return "negative";
  } else {
    return "zero";
  }
}
  • 문제 9 — 같은 타입인지 판별 
    function sameType(a, b) {
  return typeof a === typeof b;
}
  • 문제 10 — 문자열 자릿수 비교 
    function longerDigit(a, b) {
  let lenA = String(a).length;
  let lenB = String(b).length;

  if (lenA >= lenB) {
    return a;
  } else {
    return b;
  }
}
